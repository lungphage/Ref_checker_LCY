参考文献真实性自动核验工具 v5 使用说明
====================================

一、这个版本的主要特点
----------------------
1. 不需要安装 `requests`、`pandas`、`openpyxl` 等第三方库。
2. 只要 Windows 上的 Python 能正常运行 `tkinter`，就可以直接打开。
3. 支持粘贴参考文献，或导入 `txt` 文件批量核验。
4. 优先查询 Crossref，必要时自动补充 OpenAlex。
5. 批量模式会自动识别重复条目，直接复用前一次结果，减少重复查询。
6. 支持结果筛选、排序，以及按当前筛选结果导出 CSV / Excel。
7. 如果网络请求失败或结果不完整，会在结果建议中给出提示。
8. 程序出错时会写入当前目录下的 `debug_log.txt`。

二、如何运行
------------
1. 先把整个压缩包完整解压，不要直接在压缩包内双击运行。
2. 进入解压后的文件夹。
3. 推荐优先双击：

   `RUN_ME_FIRST.cmd`

它会先检查：
- 是否找得到 Python
- Python 版本是否可用
- `tkinter` 是否可用
- 然后自动启动主程序

如果你只想直接启动，也可以双击：

   `双击运行.bat`

如果双击仍然打不开，可以手动在当前目录打开 PowerShell，然后输入：

   `python -X utf8 ref_checker_gui.py`

三、基本使用方法
----------------
1. 把参考文献粘贴到输入框中，建议一行一条。
2. 如果是一整段带编号的参考文献，程序会尽量按 `[1] [2] [3]` 自动拆分。
3. 点击“开始核验”。
4. 核验完成后，可在结果区查看：
- 状态
- 相似度
- 匹配标题
- 期刊
- DOI
- 建议
5. 双击某一行，可以直接打开匹配链接或谷歌学术搜索页。

四、批量优化功能
----------------
1. 重复条目跳过：
   内容相同或核心信息相同的参考文献，会直接复用前一次结果，提升批量速度。
2. 结果筛选：
   可以按 `HIGH / MEDIUM / LOW / NOT_FOUND / 重复条目 / 疑似网络失败` 查看。
3. 结果排序：
   支持综合优先级、相似度升降序、编号升降序。
4. 筛选导出：
   可以只导出当前筛选后的结果，而不是必须导出全部。
5. 网络失败提示：
   如果 Crossref 或 OpenAlex 请求过程中失败，结果中会提示“可能不完整”。

五、状态说明
------------
HIGH：
标题高度一致，且年份或 DOI 可以互相印证，通常可以认为是真实存在的文献。

MEDIUM：
标题较接近，但作者、年份、期刊或卷期信息仍建议人工复核。

LOW：
只找到相似文献，不建议直接引用当前匹配结果。

NOT_FOUND：
没有找到可靠匹配，建议继续用标题、作者、年份到谷歌学术人工检索。

六、导出说明
------------
1. “导出 CSV”与“导出 Excel”会导出全部结果。
2. “按当前筛选导出”会只导出当前筛选后的结果。
3. CSV 使用 `utf-8-sig` 编码，Excel 可直接用常见办公软件打开。

七、如果打不开或运行异常
------------------------
请先双击：

   `环境检查.cmd`

这个脚本会检查：
- `python`
- `python --version`
- `tkinter`
- Crossref 网络访问
- OpenAlex 网络访问

如果仍有问题，请把以下内容发出来：
1. 黑窗口里的报错截图
2. 当前目录中的 `debug_log.txt` 内容

八、注意事项
------------
1. 这个工具不是直接批量爬取 Google Scholar。
2. Google Scholar 不适合高频自动化批量抓取，容易触发验证码或访问限制。
3. 本工具的目标是：
- 验证参考文献是否真实存在
- 返回更可靠的标题、年份、期刊、作者、DOI 和链接
4. 对于硕博论文、会议摘要、中文数据库独有文献，Crossref / OpenAlex 可能覆盖不全，仍需要人工复核。

九、关于打包 EXE 体积
--------------------
1. 默认的 `PyInstaller --onefile` 会把 Python 运行时和依赖全部塞进一个 EXE，所以文件通常会比较大。
2. 现在附带的 `打包成EXE.cmd` 默认使用的是更适合本工具的 `--onedir` 模式。
3. 这种模式下：
- `exe` 主文件会明显更小
- 但需要连同整个 `dist\ReferenceChecker` 文件夹一起分发
4. 如果以后你一定要单文件 EXE，可以改回 `--onefile`，但体积通常会明显增大，这是正常现象。
5. 如果打包时报 `No module named 'pkg_resources'`，通常是旧版 PyInstaller 遇到了新版 `setuptools`。当前打包脚本已经会自动升级 PyInstaller，并把 `setuptools` 调整到兼容范围。
6. 不建议使用 `Python 3.15 alpha/beta` 这类预发布版本打包。当前工具更建议使用稳定版 `Python 3.12 / 3.13 / 3.14`。
